Security

Threats and social engineering, malware types and removal, authentication and access control, wireless security, and hardening workstations & SOHO networks.

2.1 Social engineering & threats

AttackWhat it is
PhishingFraudulent email/message luring credentials. Spear phishing = targeted; whaling = execs.
Vishing / SmishingPhishing via voice call / SMS.
Shoulder surfingWatching someone enter credentials.
Tailgating / PiggybackingFollowing an authorized person through a secure door.
Impersonation / PretextingPretending to be someone to gain trust/access.
Dumpster divingRecovering sensitive info from the trash.
Evil twinRogue AP mimicking a legitimate Wi-Fi network.
  • Network attacks: DoS/DDoS, on-path (MITM), spoofing, brute force, dictionary, zero-day, SQL injection, XSS.
  • Insider threat and non-compliant systems are vulnerabilities too.

2.2 Malware & removal

TypeBehavior
VirusAttaches to files; needs a host & user action to spread.
WormSelf-replicates across networks; no host needed.
TrojanDisguised as legitimate software.
RansomwareEncrypts data and demands payment.
Spyware / KeyloggerSecretly records activity/keystrokes.
RootkitHides deep in the OS; very hard to detect/remove.
CryptominerHijacks resources to mine cryptocurrency.
Malware removal — know the 7 steps in order: (1) Investigate & verify symptoms, (2) Quarantine the system, (3) Disable System Restore (Windows), (4) Remediate (update & scan, safe mode/pre-installation env), (5) Schedule scans & run updates, (6) Re-enable System Restore & create a restore point, (7) Educate the end user.

2.3 Authentication & access control

  • MFA factors: something you know (password), have (token/phone), are (biometric), and somewhere you are/do.
  • Least privilege, principle of least privilege, ACLs, user/group permissions.
  • Directory/AAA: Active Directory, Kerberos, RADIUS/TACACS+, SSO.
  • Hard token vs soft token, smart cards, biometrics.

2.4 Wireless & SOHO security

ProtocolStrength
WEPBroken — never use.
WPA (TKIP)Weak — avoid.
WPA2 (AES/CCMP)Strong — minimum acceptable.
WPA3 (SAE)Strongest — current best practice.
  • Harden a SOHO router: change default admin credentials, update firmware, disable WPS, change SSID, enable WPA2/WPA3, disable remote management, set up a guest network, use a strong passphrase.
  • Firewall: enable, allow/deny by port; content filtering; MAC filtering (weak control); port forwarding vs DMZ.

2.5 Workstation & data security

  • Best practices: strong password policy, account lockout, screen lock/timeout, least privilege, disable AutoRun, restrict logon times, change default accounts.
  • Encryption: BitLocker (whole drive), EFS (files), TPM-backed keys.
  • Data destruction: physical (shred, drill, degauss, incinerate) vs sanitization (secure erase, low-level format, wipe). Certificate of destruction for compliance.
  • Antivirus/anti-malware, host firewall, patches, secure DNS.
On the exam: Reusing a drive → wipe/secure erase. Disposing of a drive with sensitive data → physical destruction (shred/degauss/drill).
