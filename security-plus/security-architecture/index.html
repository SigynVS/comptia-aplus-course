Security+ · SY0-701 · Domain 3.0 (18%)

Security Architecture

Designing secure systems: architecture models, secure network design, data protection, and resilience.

3.1 Architecture models

  • Cloud (responsibility matrix), serverless, microservices, IaC (infrastructure as code).
  • On-prem vs cloud vs hybrid; virtualization/containers; IoT/ICS/SCADA, embedded, RTOS.
  • Zero Trust architecture: control plane vs data plane; policy enforcement points.

3.2 Secure network design

ConceptNotes
Segmentation / VLAN / DMZLimit blast radius; isolate public-facing services.
Firewalls (NGFW/UTM/WAF)Layer 3–7 filtering; WAF protects web apps.
IDS/IPSDetect / actively block intrusions.
Proxy / load balancerMediation, caching, scaling, availability.
VPN / SASE / SD-WANSecure remote and site connectivity.
Port security / 802.1X / NACControl device access at the edge.

Design principles: fail-open vs fail-closed, least privilege, defense in depth, secure defaults.

3.3 Data protection

  • Data states: at rest, in transit, in use — protect each (encryption, TLS, enclaves).
  • Classifications: public, sensitive, confidential, restricted; PII/PHI.
  • Techniques: encryption, tokenization, masking, hashing, obfuscation, DLP.
  • Geographic/sovereignty considerations and data lifecycle.
On the exam: Tokenization replaces data with a non-sensitive token (reversible via a vault); masking hides part of the data (e.g., ****1234).

3.4 Resilience & recovery

  • High availability: clustering, load balancing, geographic redundancy, multi-cloud.
  • Backups: types, frequency, encryption, offsite, 3-2-1; test restores.
  • Power: UPS, generators, dual supplies, PDUs.
  • Continuity: RTO/RPO, hot/warm/cold sites, tabletop exercises, failover testing.
Tip: Capacity planning + tested backups + redundant power are the unglamorous core of resilience.
