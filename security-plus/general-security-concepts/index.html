General Security Concepts

The vocabulary the whole exam is built on: the CIA triad, the categories and types of security controls, AAA, zero trust, core cryptography and PKI, and change management.

1.1 The CIA triad (and friends)

  • Confidentiality — only authorized parties can read data (encryption, access control).
  • Integrity — data is unaltered and trustworthy (hashing, digital signatures).
  • Availability — systems/data are accessible when needed (redundancy, backups).
  • Non-repudiation — a party cannot deny an action (digital signatures, logging).
  • Authentication, Authorization, Accounting (AAA) — prove identity, grant access, record activity.
On the exam: Map a scenario to the right pillar. Tampered data = integrity; a DDoS knocking a site offline = availability; a leak = confidentiality.

1.3 Zero Trust

  • Core idea: "never trust, always verify" — no implicit trust based on network location.
  • Control plane: adaptive identity, policy engine, policy administrator, policy decision point.
  • Data plane: subject/system, policy enforcement point.
  • Concepts: least privilege, microsegmentation, continuous verification.

1.4 Physical security

  • Barriers: bollards, fencing, access control vestibule (mantrap).
  • Detection: sensors (infrared, pressure, microwave, ultrasonic), cameras, guards, lighting.
  • Honeypots/honeynets/honeyfiles and honeytokens — deception/disruption to detect attackers.

1.5 Cryptographic solutions

ConceptNotes
SymmetricOne shared key (AES). Fast; key distribution is the challenge.
AsymmetricPublic/private key pair (RSA, ECC). Solves key exchange; slower.
HashingOne-way integrity check (SHA-256). Not encryption.
Digital signatureHash encrypted with the sender's private key → integrity + non-repudiation.
SaltingRandom data added before hashing passwords to defeat rainbow tables.
TPM / HSMHardware that stores keys securely (TPM on a device; HSM as an appliance).
Key escrow / KMSSecure key storage/recovery and lifecycle management.

PKI essentials

  • CA issues certificates; RA verifies identities; certificates bind a public key to an identity.
  • CRL and OCSP check whether a certificate is revoked.
  • Root of trust, intermediate CAs, certificate chaining, and wildcard certs.
On the exam: Hashing proves integrity (not confidentiality). A digital signature = sign the hash with your private key; verify with your public key.

1.6 Change management

  • Business processes: approval, ownership, stakeholders, impact analysis, test results, backout plan, maintenance window.
  • Technical implications: allow/deny lists, restricted activities, downtime, service/app restarts, legacy apps, dependencies.
  • Documentation: update diagrams/policies; version control.
Tip: Change management overlaps heavily with A+ Operational Procedures — approvals + backout plans are universal.
