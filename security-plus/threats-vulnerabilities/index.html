Security+ · SY0-701 · Domain 2.0 (22%)

Threats, Vulnerabilities & Mitigations

Who attacks, how they attack, where systems are weak, and how to reduce the risk.

2.1 Threat actors

ActorTraits / motivation
Nation-state / APTHighly resourced; espionage, sabotage. Advanced Persistent Threat.
Organized crimeFinancial gain (ransomware, fraud).
HacktivistIdeological/political motivation.
Insider threatEmployee/contractor; access + intent or negligence.
Unskilled (script kiddie)Uses prebuilt tools.
Shadow ITUnsanctioned systems/services created internally.

Attributes: internal/external, resources/funding, sophistication. Motivations: data exfiltration, financial, espionage, disruption, revenge.

2.2 Social engineering

  • Phishing (email), vishing (voice), smishing (SMS), spear/whaling (targeted).
  • Pretexting, impersonation, BEC (business email compromise), pharming, watering hole.
  • Principles of influence: authority, urgency, scarcity, social proof, intimidation, familiarity, trust.

2.3 Malware & attack types

TypeBehavior
RansomwareEncrypts data, demands payment.
Worm / Virus / TrojanSelf-spreading / host-bound / disguised.
Rootkit / Logic bombStealthy persistence / triggered payload.
Spyware / KeyloggerSurveillance / keystroke capture.
  • App attacks: injection (SQLi), XSS, CSRF, buffer overflow, privilege escalation, replay.
  • Crypto/password: brute force, dictionary, spraying, birthday, downgrade, collision.
  • Network: DDoS, on-path, DNS attacks, zero-day.
On the exam: Zero-day = no patch exists yet. Password spraying tries a few common passwords across many accounts to avoid lockout.

2.4 Vulnerabilities

  • Application: memory injection, buffer overflow, race conditions (TOC/TOU), insecure APIs.
  • OS/web/cloud/virtualization: misconfigurations, VM escape, supply-chain, EOL/legacy systems.
  • Mobile: sideloading, jailbreak/root. Hardware: firmware, EOL, end-of-support.

2.5 Mitigation techniques

  • Segmentation, access control (ACL/least privilege), application allow lists, isolation/sandboxing.
  • Patching, hardening, encryption, monitoring, configuration enforcement, decommissioning.
  • Removal of unnecessary software/services; default credential changes.
Tip: "Defense in depth" = layered mitigations so no single failure is catastrophic.
