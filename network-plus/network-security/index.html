Network Security

Core security concepts, common network attacks, device hardening, and secure remote access.

4.1 Security concepts

  • CIA triad, defense in depth, least privilege, zero trust.
  • AAA, RADIUS/TACACS+, 802.1X port-based auth, MFA.
  • Network segmentation: VLANs, DMZ/screened subnet, NAC (network access control).

4.2 Common attacks

AttackWhat it does
DoS / DDoSOverwhelms a service to deny availability.
On-path (MITM)Intercepts/alters traffic between two parties.
ARP spoofing/poisoningMaps the attacker's MAC to a victim IP on the LAN.
DNS poisoningReturns false DNS answers to redirect victims.
VLAN hoppingReaches a VLAN the attacker shouldn't access.
MAC floodingOverflows the switch CAM table to force flooding.
Rogue AP / evil twinUnauthorized AP impersonating a legit network.

4.3 Hardening

  • Disable unused ports/services; change default credentials; update firmware.
  • Port security, DHCP snooping, dynamic ARP inspection, BPDU guard, root guard.
  • Wireless: WPA2/WPA3, disable WPS, strong PSK or 802.1X (enterprise).
  • Use ACLs, host/network firewalls, IDS/IPS, and private VLANs.
On the exam: DHCP snooping blocks rogue DHCP servers; dynamic ARP inspection defeats ARP spoofing.

4.4 Remote access

  • VPN: site-to-site vs client (remote-access); IPsec (tunnel/transport), SSL/TLS VPN, split vs full tunnel.
  • SSH (22) for secure CLI; RDP (3389); out-of-band management (console).
  • SASE/SSE and ZTNA for modern remote-access security.
Tip: Prefer SSH over Telnet and a VPN over exposing services directly — encryption in transit is the theme.
