Network Troubleshooting

The methodology, cabling and connectivity faults, performance issues, and the tools that find them.

5.1 The troubleshooting methodology

  1. Identify the problem (gather info, question users, duplicate if possible).
  2. Establish a theory of probable cause (question the obvious).
  3. Test the theory to determine cause.
  4. Establish a plan of action and identify effects.
  5. Implement the solution or escalate.
  6. Verify full functionality and implement preventive measures.
  7. Document findings, actions, and outcomes.
On the exam: Same 7-step flow as A+. Memorize the order — questions often ask "what's the NEXT step."

5.2 Cabling & physical issues

SymptomLikely cause
No link lightBad cable, wrong port, NIC, or speed/duplex.
Intermittent connectivityLoose connector, EMI, damaged cable.
Slow throughputDuplex mismatch, saturation, bad cable, CRC errors.
Crosstalk / attenuationCable too long (>100 m), poor termination, interference.
Wrong pinoutT568A/B mismatch; need straight-through vs crossover.

5.3 Connectivity & addressing

  • 169.254.x.x (APIPA) → no DHCP. Duplicate IP → conflict.
  • Wrong gateway/DNS/subnet mask → can't reach remote networks.
  • DNS issues → can ping IP but not name; routing loops → high hop counts/TTL expiry.
