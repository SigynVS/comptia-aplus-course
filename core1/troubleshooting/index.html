Core 1 · 220-1101 · Domain 5.0 (29%)

Hardware & Network Troubleshooting

The largest domain. Master the methodology first, then the symptom-to-cause patterns for devices, displays, storage, and networks.

5.1 The CompTIA troubleshooting methodology

This 6-step process appears on both exams. Know the order.

  1. Identify the problem — question the user, back up data, note environment/recent changes.
  2. Establish a theory of probable cause (question the obvious; consider multiple causes).
  3. Test the theory to determine cause. If confirmed, plan next steps; if not, establish a new theory or escalate.
  4. Establish a plan of action to resolve the problem and implement the solution.
  5. Verify full system functionality and, if applicable, implement preventive measures.
  6. Document findings, actions, and outcomes.
On the exam: "Always consider corporate policies, procedures, and impacts before implementing changes." And back up data as part of identifying the problem.

5.2 Motherboard, CPU, RAM & power symptoms

SymptomLikely cause
No power / no POSTPSU, power button, RAM seating, shorted board.
POST beeps / no displayRAM or GPU not seated; check beep codes.
Intermittent shutdownsOverheating (dust, failed fan, dried thermal paste) or failing PSU.
Burning smell / swollen capacitorsFailing PSU or motherboard — replace.
Continuous reboots / BSOD / GSODRAM, drivers, overheating, failing drive.
Wrong date/timeDead CMOS battery.
Loud whining / grindingFailing fan or HDD bearings.

5.3 Storage & RAID symptoms

  • Clicking / grinding noises = mechanical HDD failure → back up immediately.
  • S.M.A.R.T. failure warning = drive predicting failure.
  • Bootable device not found = boot order, disconnected drive, or dead drive.
  • RAID not found / RAID stops = failed member disk or controller; rebuild from parity/mirror.
  • Extended read/write times, data loss/corruption = failing drive or cabling.

5.4 Display symptoms

SymptomLikely cause
Dim imageBacklight/inverter, brightness setting, power source (laptop).
FlickeringLoose cable, refresh rate, failing backlight.
Dead pixelsManufacturing defect (stuck/dead).
Burn-in / image persistenceOLED/plasma static content over time.
Incorrect color / distorted imageCable, GPU, drivers, resolution mismatch.
No image / external display neededUse the laptop's display toggle / detect external.

5.5 Mobile device symptoms

  • Poor battery life / swollen battery — replace; a swollen battery is a safety hazard, stop use.
  • Overheating — close apps, check charging, environment.
  • Won't charge / slow charge — cable, port debris, charger wattage.
  • Random reboots, app crashes, slow performance — update, clear cache, factory reset as last resort.

5.6 Network troubleshooting

SymptomLikely cause / tool
No connectivityCable, NIC, switch port, IP config. ping, link lights.
APIPA (169.254.x.x)No DHCP — check DHCP service/cable.
Intermittent / slow / high latencyInterference, duplex mismatch, saturation, bad cable.
Limited connectivityWrong gateway/DNS; verify with ipconfig.
Weak Wi-Fi signalDistance, interference, antenna; relocate AP / change channel.
Port flappingBad cable/NIC or switch port cycling up/down.

Tools: cable tester, loopback plug, toner probe, punchdown tool, crimper, Wi-Fi analyzer, and software like ping, ipconfig, tracert, nslookup.

On the exam: A swollen battery = stop using the device and replace it. Don't puncture it.
