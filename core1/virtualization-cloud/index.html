Virtualization & Cloud Computing

Cloud service and deployment models, shared/metered resources, and how virtual machines and hypervisors work on the client side.

4.1 Cloud concepts

Service models

ModelYou manageExample
IaaS (Infrastructure)OS, apps, data (provider gives VMs/storage/network)AWS EC2
PaaS (Platform)Your app & data (provider gives the runtime)App Engine, Heroku
SaaS (Software)Just use it (provider manages everything)Microsoft 365, Gmail

Deployment models

  • Public — shared provider infrastructure (AWS, Azure).
  • Private — dedicated to one organization.
  • Hybrid — mix of public and private.
  • Community — shared by organizations with common needs.

Cloud characteristics

  • Shared resources (multitenancy), rapid elasticity (scale up/down on demand).
  • Metered / measured utilization — pay for what you use.
  • High availability, on-demand self-service, broad network access.
  • File synchronization across devices (cloud storage).
  • Desktop virtualization: VDI in the cloud or on-prem.
On the exam: Match the aaS term to "how much you manage." SaaS = least for you; IaaS = most. PaaS sits in the middle (you bring the app).

4.2 Client-side virtualization

Hypervisors

  • Type 1 (bare-metal) — runs directly on hardware (ESXi, Hyper-V server). Used for data-center virtualization.
  • Type 2 (hosted) — runs on top of a host OS (VirtualBox, VMware Workstation). Used on a technician's workstation.

Resource requirements

  • Adequate CPU (with virtualization extensions: Intel VT-x / AMD-V enabled in BIOS/UEFI), RAM, storage, and networking.
  • Each VM needs its own allocated resources; over-provisioning starves the host.

Purposes & security

  • Sandbox/test environments, running legacy or cross-platform apps, developer testing.
  • Security: isolate malware analysis in a VM; beware VM escape and the need to patch the hypervisor.
  • Resource pooling and snapshots (roll back a VM to a known state).
Tip: If a VM won't start with a virtualization error, enable VT-x / AMD-V in the firmware (BIOS/UEFI).
